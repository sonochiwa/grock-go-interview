# Указатели

## Обзор

Указатель хранит адрес переменной в памяти. В Go указатели безопасны: нет арифметики указателей (кроме unsafe), есть garbage collector. Понимание указателей критично для работы с методами, интерфейсами и оптимизации.

## Концепции

### Основы

```go
x := 42
p := &x      // p — указатель на x, тип *int
fmt.Println(*p) // 42 — разыменование
*p = 100
fmt.Println(x)  // 100 — изменили x через указатель

var nilPtr *int // nil — нулевой указатель
// *nilPtr      // panic: nil pointer dereference
```

### Указатель vs Значение

```go
// Передача по значению — копирование
func double(x int) {
    x *= 2 // изменяем копию
}

// Передача по указателю — изменение оригинала
func doublePtr(x *int) {
    *x *= 2 // изменяем оригинал
}

n := 10
double(n)     // n == 10 (не изменился)
doublePtr(&n) // n == 20
```

### new() vs &

```go
// Оба создают указатель на zero value
p1 := new(int)    // *int, значение 0
p2 := &int{}      // НЕ работает для базовых типов!

// Для структур оба варианта OK
s1 := new(MyStruct)    // *MyStruct, все поля zero
s2 := &MyStruct{}      // *MyStruct, все поля zero
s3 := &MyStruct{X: 1}  // *MyStruct, с инициализацией
```

### Когда использовать указатели

| Используй указатель | Используй значение |
|---|---|
| Нужно изменить объект | Маленькая структура (< ~64 байт) |
| Большая структура (избегаем копирования) | Неизменяемые данные |
| Нужно выразить "отсутствие" (nil) | Базовые типы (int, bool, string) |
| Реализуешь интерфейс с pointer receiver | Slices, maps, channels (уже ссылочные) |

## Под капотом: стек vs куча

### Escape analysis

Компилятор Go решает, где разместить переменную — на **стеке** (быстро, автоматическая очистка) или в **куче** (медленнее, нужен GC):

```go
func stackAlloc() int {
    x := 42  // остаётся на стеке
    return x  // возвращается копия
}

func heapAlloc() *int {
    x := 42   // ESCAPE: переменная "убегает" в кучу
    return &x  // возвращается указатель — x должен жить после возврата
}
```

Проверка escape analysis:
```bash
go build -gcflags="-m" main.go
# ./main.go:10:2: moved to heap: x
```

### Что вызывает escape в кучу

1. Возврат указателя на локальную переменную
2. Присвоение указателя глобальной переменной
3. Сохранение в интерфейс (interface{})
4. Замыкание захватывает переменную
5. Слайс, растущий за пределы стека
6. Отправка указателя в канал

```go
// Пример: интерфейс вызывает escape
func printAny(v any) { fmt.Println(v) }

func main() {
    x := 42
    printAny(x) // x escapes to heap — упаковывается в interface{}
}
```

## unsafe.Pointer

Для низкоуровневых операций (обычно не нужно, но спрашивают):

```go
import "unsafe"

// Получить размер и выравнивание
fmt.Println(unsafe.Sizeof(int64(0)))  // 8
fmt.Println(unsafe.Alignof(int64(0))) // 8

// Конвертация между указателями разных типов
var x float64 = 3.14
bits := *(*uint64)(unsafe.Pointer(&x))
// bits — это IEEE 754 представление 3.14

// unsafe.Pointer правила (в документации):
// 1. *T → unsafe.Pointer — OK
// 2. unsafe.Pointer → *T — OK
// 3. unsafe.Pointer → uintptr — OK (для арифметики)
// 4. uintptr → unsafe.Pointer — ОПАСНО (GC может переместить объект)
```

## Частые вопросы на собеседованиях

**Q: Есть ли в Go арифметика указателей?**
A: Нет, только через unsafe.Pointer → uintptr. Обычный код не должен этого делать.

**Q: Что произойдёт при разыменовании nil указателя?**
A: panic: runtime error: invalid memory address or nil pointer dereference

**Q: Где размещается переменная — на стеке или в куче?**
A: Решает компилятор через escape analysis. Если указатель на переменную может жить дольше фрейма стека — переменная уходит в кучу.

**Q: Почему передача большой структуры по указателю быстрее?**
A: Указатель — 8 байт (на 64-bit). Копирование структуры в 1КБ — 1024 байта. Но для маленьких структур (< ~64 байт) копирование может быть быстрее из-за cache locality и отсутствия indirection.

## Подводные камни

1. **nil pointer в интерфейсе**: интерфейс с nil указателем внутри != nil интерфейс (подробнее в разделе интерфейсов)

2. **Указатель на элемент цикла (до Go 1.22)**:
```go
// BUG до Go 1.22: все указатели указывают на последний элемент
var ptrs []*int
for _, v := range []int{1, 2, 3} {
    ptrs = append(ptrs, &v) // v переиспользуется!
}
// В Go 1.22+ это исправлено — каждая итерация создаёт новую v
```

3. **Не используй указатели на слайсы/мапы/каналы** — они уже содержат указатель внутри:
```go
// Плохо
func process(s *[]int) { ... }
// Хорошо (если не нужно менять сам слайс, а не элементы)
func process(s []int) { ... }
// Исключение: если нужно изменить длину/ёмкость слайса вызывающего кода
func appendTo(s *[]int, val int) { *s = append(*s, val) }
```
