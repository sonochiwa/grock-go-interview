# 02 — Интерфейсы в Go

Интерфейсы — один из ключевых механизмов абстракции в Go. В отличие от большинства ООП-языков, Go использует **неявную реализацию** (structural typing / duck typing), что кардинально меняет подход к проектированию.

## Ключевые принципы

1. **Интерфейсы неявные** — тип реализует интерфейс, если имеет все нужные методы. Никаких `implements`.
2. **Держите интерфейсы маленькими** — чем меньше методов, тем больше типов могут удовлетворить интерфейс.
3. **Определяйте интерфейсы на стороне потребителя**, а не на стороне реализации.
4. **Интерфейс — это пара указателей** (тип + данные), что влияет на производительность и аллокации.
5. **`nil`-интерфейс и интерфейс, хранящий `nil`** — это разные вещи. Классическая ловушка.
