# 03 — Обработка ошибок в Go

Обработка ошибок — одна из самых обсуждаемых и самых важных тем в Go. Язык сознательно отказался от исключений в пользу явных возвращаемых значений. Это решение влияет на архитектуру, читаемость и надёжность кода. На собеседованиях вопросы об ошибках проверяют понимание идиоматического Go, паттернов оборачивания, стратегий обработки по слоям и внутренней механики `errors.Is` / `errors.As`.

## Ключевые принципы

1. **Ошибки — это значения**. Их можно хранить, передавать, оборачивать, программировать.
2. **Не игнорируйте ошибки**. `_ = doSomething()` — путь к неуловимым багам в production.
3. **Оборачивайте с контекстом**. `fmt.Errorf("загрузка пользователя %d: %w", id, err)` — дайте будущему себе понять, где сломалось.
4. **Обрабатывайте ошибку один раз**. Не логируйте И не возвращайте — выберите что-то одно.
5. **Используйте `errors.Is` / `errors.As`** вместо `==` и type assertion — они работают с обёрнутыми ошибками.
