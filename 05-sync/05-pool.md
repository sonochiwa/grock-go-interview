# sync.Pool

## Обзор

Pool — кеш объектов для переиспользования. Снижает нагрузку на GC. Объекты могут быть удалены GC в любой момент.

## Концепции

```go
var bufPool = sync.Pool{
    New: func() any {
        return new(bytes.Buffer)
    },
}

func process(data []byte) string {
    buf := bufPool.Get().(*bytes.Buffer)
    defer func() {
        buf.Reset() // ОБЯЗАТЕЛЬНО очисти перед возвратом!
        bufPool.Put(buf)
    }()

    buf.Write(data)
    return buf.String()
}
```

### Когда использовать

- Частые аллокации одного типа объектов (буферы, слайсы)
- Объекты дорого создавать
- Аллокации видны в pprof (alloc_objects)

### Когда НЕ использовать

- Для пула соединений (GC может очистить!) — используй database/sql.DB
- Для маленьких объектов (overhead pool > выигрыш)

### Реальные примеры в стандартной библиотеке

- `fmt` — переиспользует буферы для Sprintf
- `encoding/json` — переиспользует encoder state
- `net/http` — переиспользует буферы для чтения

## Частые вопросы на собеседованиях

**Q: Может ли GC удалить объекты из Pool?**
A: Да. Pool очищается при каждом цикле GC. Это НЕ объектный пул с гарантированным размером.

**Q: Нужно ли очищать объект перед Put?**
A: Да! Иначе следующий Get получит объект с "грязным" состоянием.
