# Happens-Before

## Обзор

Happens-before — частичный порядок между событиями в программе. Если событие A happens-before B, то эффекты A гарантированно видны B.

## Концепции

### Внутри одной горутины

Всё последовательно. Компилятор может переупорядочить инструкции, но наблюдаемое поведение как в исходном коде.

```go
// Компилятор гарантирует: a = 1 выполнится до b = 2 (с точки зрения горутины)
a = 1
b = 2
```

### Между горутинами — NO GUARANTEES без синхронизации!

```go
var a, b int

go func() {
    a = 1
    b = 2
}()

// Другая горутина может увидеть:
// a=0, b=0  (ещё не записано)
// a=1, b=0  (частично)
// a=0, b=2  (переупорядочивание!)
// a=1, b=2  (всё записано)
```

### Правила happens-before в Go

1. **init()** → **main()**: init пакета happens-before main
2. **go statement** happens-before начало горутины
3. **Channel send** happens-before соответствующий receive завершится
4. **close(ch)** happens-before receive, возвращающий zero value
5. **Unbuffered receive** happens-before send завершится
6. **Mutex.Unlock()** happens-before следующий Lock()
7. **Once.Do(f)** вызов f happens-before любой Do возвращается
8. **Atomic** операции обеспечивают sequential consistency

## Частые вопросы на собеседованиях

**Q: Что такое happens-before?**
A: Формальное отношение порядка. Если A happens-before B, то все эффекты A видны B. Без happens-before — никаких гарантий.

**Q: Может ли компилятор переупорядочить инструкции?**
A: Да, и CPU тоже может. Единственное ограничение — наблюдаемое поведение внутри горутины не меняется. Между горутинами порядок гарантируется только через sync points.
